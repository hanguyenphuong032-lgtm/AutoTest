1. Overview
Cucumber
→ Bạn viết kịch bản .feature kiểu:
Given / When / Then.

Serenity BDD
→ Chạy test + generate report HTML đẹp, log từng bước rõ ràng.

Screenplay pattern (trong Serenity)
→ Thay vì viết test theo kiểu “script” (step 1, step 2…), bạn tổ chức theo:

Actor: Người dùng (User)

Task: Việc actor làm (Login, Search,…)

Question: Điều cần kiểm tra (SeeWelcomeMessage,…)

Ability: Actor có khả năng gì (BrowseTheWeb,…)

2. Cấu trúc project (Maven)
Ví dụ 1 project web test:

src
 └── test
     ├── java
     │   ├── starter
     │   │   ├── navigation   (Tasks, Questions, Abilities…)
     │   │   ├── ui           (Page / Target elements)
     │   │   └── stepdefinitions (Step defs cho Cucumber)
     │   └── runner
     │       └── CucumberTestSuite.java
     └── resources
         └── features
             └── login
                 └── login.feature

3. pom.xml tối thiểu (Maven)
<dependencies>
    <!-- Serenity với Cucumber -->
    <dependency>
        <groupId>net.serenity-bdd</groupId>
        <artifactId>serenity-cucumber</artifactId>
        <version>4.1.20</version>
    </dependency>

    <!-- Screenplay core -->
    <dependency>
        <groupId>net.serenity-bdd</groupId>
        <artifactId>serenity-screenplay</artifactId>
        <version>4.1.20</version>
    </dependency>

    <!-- Screenplay WebDriver -->
    <dependency>
        <groupId>net.serenity-bdd</groupId>
        <artifactId>serenity-screenplay-webdriver</artifactId>
        <version>4.1.20</version>
    </dependency>

    <!-- JUnit (tùy bạn dùng JUnit4 hay 5) -->
    <dependency>
        <groupId>junit</groupId>
        <artifactId>junit</artifactId>
        <version>4.13.2</version>
        <scope>test</scope>
    </dependency>

    <!-- Selenium (Serenity sẽ wrap) -->
    <dependency>
        <groupId>org.seleniumhq.selenium</groupId>
        <artifactId>selenium-java</artifactId>
        <version>4.23.0</version>
    </dependency>
</dependencies>

<build>
    <plugins>
        <plugin>
            <groupId>net.serenity-bdd.maven.plugins</groupId>
            <artifactId>serenity-maven-plugin</artifactId>
            <version>4.1.20</version>
            <configuration>
                <tags>@regression</tags>
            </configuration>
        </plugin>
    </plugins>
</build>
